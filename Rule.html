ArgumentRule~SetupCallback RegExp Rule~Token Rule~Composition

Rule constructor takes unlimited number of arguments.

CompositionArray.<String>

Rules can compose other rules. The composition is done by specifying an array of Rule namespaces. The namespaces given can be without parent prefix if specifying a sibling Rule. All compositions happen inline, this means the definition for the Rule being composed needs to happen first.

Each composed Rule's children, actions Composing a rule takes all that rules children and actions

Making a rule Rule~Options.exclusive and then composing it in multiple places is the intent, however you may use composition however you like.
